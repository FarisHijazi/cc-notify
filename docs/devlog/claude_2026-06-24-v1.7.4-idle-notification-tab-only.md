# v1.7.4 — idle Notification is tab-only (no banner); token docs synced

Follow-up to @claude_2026-06-24-v1.7.3-askuserquestion-and-100-token.md.

## 1. Disable the ~60s idle notification banner (keep tab sync)

Claude Code fires a `Notification` (`idle_prompt`, "Claude is waiting for your
input") ~60s after a turn ends if untouched. It's low-signal and noisy, and the
instant `Stop` ping already covers "done". User wants it to update the tab status
only, no banner.

**Change** (`hooks/cc-notify.sh`): the idle case in the notification branch now sets
`notif_tab_only=1`; just before spawning the banner bg worker, `[ -n "$notif_tab_only" ] && exit 0`.
The tab write (`cc_write_tab`, → ❓) happens earlier in the script and is unaffected,
so the tab still syncs. **Permission** prompts (`permission_prompt` → 🔐) and the
generic fallback (🔔) still banner.

Tested: `idle_prompt` → 0 bg-workers spawned; `permission_prompt` and an unknown
type → bg-worker spawned. (Hardcoded, not a toggle file — YAGNI; the user wants it
off, period.)

## 2. Token docs synced

- Global `~/.claude/cc-notify-tokens.md` and repo `docs/cc-notify-tokens.md` both got
  the rule: the token emojis (🚨 💯 ✅ ❌ 🚫 🙋 👍 👎 🏃 ℹ️) appear ONLY as the single
  trailing token — never inline, in bullets, or as heading checkmarks (use plain text
  or `✓`, which is not in the set). Came from the user noticing decorative `✅`s in a
  prior reply.
- The repo copy was ALSO missing the `💯` token entirely (the global had it) — added
  it as #2, noting cc-notify renders it as `💯✅`.
