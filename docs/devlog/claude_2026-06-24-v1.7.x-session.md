## Architecture / where things live
See @../../CLAUDE.md (kept current) — `hooks/cc-lib.sh` is the shared lib (detection, emoji maps, tab writes, token parser); `hooks/cc-capture-window.sh` is the status updater for the 7 non-banner hooks; `hooks/cc-notify.sh` is the banner path; `editor-extension/` does pane focus + file-watch tab rename.

## CRITICAL gotcha for the next session
- **Bump `.claude-plugin/plugin.json` `version` on EVERY change**, else `/plugin update` sees "already at X" and silently no-ops → installed sessions run a frozen stale cache. This bit us hard (frozen 1.7.0). Alternative not yet done: drop `version` from plugin.json so the marketplace tracks the commit SHA (auto-update every push).
- **Distribution = single source of truth**: develop in the `cc-notify` repo, push to `main`. The marketplace `claude-plugins` references it externally (`source: {github, repo: FarisHijazi/cc-notify}`); the old bundled `plugins/cc-notify/` copy is deleted. Do NOT hand-patch `~/.claude/plugins/cache|marketplaces` for distribution — only for testing the current session (autoUpdate/`/plugin update` overwrite them).
- Editor `extension.js` changes need a **window reload** to take effect (file changes are live via symlink, loaded code is not).
- `hooks.json` changes only apply to **newly started** `claude` sessions.
