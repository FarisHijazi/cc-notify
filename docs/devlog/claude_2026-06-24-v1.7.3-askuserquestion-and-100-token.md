# v1.7.3 — 🔀 AskUserQuestion status; 💯 token recognized (shown as 💯✅)

Follow-up to @claude_2026-06-24-v1.7.2-focus-self-heal.md. Three user asks.

## 1. AskUserQuestion (multiple-choice TUI menu) → 🔀

**Finding:** AskUserQuestion does NOT fire a `Notification` hook. `notiftypes.log`
only ever shows `idle_prompt` and `permission_prompt`. But it IS a tool, so it
fires `PreToolUse` with `tool_name: "AskUserQuestion"` (and the menu blocks until
the user answers, so the status persists until its `PostToolUse`).

**Change** (`hooks/cc-capture-window.sh`): added `tool_name` to the hook's JSON
parse, and `PreToolUse` now routes `AskUserQuestion → options` (🔀); every other
tool stays `running` (⏳). PostToolUse/SubagentStop stay ⏳. New `options) 🔀` entry
in `cc_status_emoji`. Tab-only (cc-capture-window never banners) — fits the request.

## 2. "What does the hourglass mean?"

⏳ = Claude is **actively working** (a turn is in progress). It does NOT mean done.
"Done" is the outcome token (💯✅/✅/❌/…) or ℹ️/👀 when there's no token. No code
change — clarified in README + CLAUDE.md. (If ⏳ ever lingers after done, that's the
stale-tab case the v1.7.2 focus-reapply already self-heals.)

## 3. 💯 token → display as 💯✅

**Bug:** `💯` is priority-2 in `cc-notify-tokens.md` (Claude emits it for "all tasks
done, nothing left"), but `cc_last_status_token`'s detection array was missing it,
so a `💯` ending silently fell through to ℹ️.

**Change** (`hooks/cc-lib.sh`):
- Added `💯` to the `cc_last_status_token` array → now detected.
- New `complete) 💯✅` in `cc_status_emoji`; `cc-notify.sh` Stop branch maps the
  `💯` token → `cc_status_emoji complete` (displays the two-glyph 💯✅), subtitle
  "All tasks complete".
- `cc_set_status`'s leading-emoji strip is now a **run** (`(?:…)+`) and includes
  `💯`/`🔀`, so the two-glyph 💯✅ prefix is replaced cleanly when a later cheap
  swap (PreToolUse etc.) sets ⏳. The color circle isn't in the set, so the run
  stops at `<color> <name>` and never eats them. (Caught a bug here: an apostrophe
  in a code comment closed the bash single-quoted `node -e` block — comments inside
  these inline node scripts must be apostrophe-free.)
