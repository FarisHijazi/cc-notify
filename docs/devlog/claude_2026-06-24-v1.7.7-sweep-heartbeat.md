# v1.7.7 — 30s heartbeat sweep (hook-independent)

## Problem

v1.7.6 wired `cc_trigger_sweep` into hooks, but the user still saw no sweeps:
- Hook scripts only update on **new sessions started after a plugin redeploy** — the
  running sessions still had the old hooks (no trigger).
- Even once deployed, triggers only fire on settled **events** (Stop/Notification/
  SessionStart/SessionEnd), so a long-idle background tab could still drift.

User: "make it fire every 30s even if there was no hook to trigger it, but those 30s
fires should be queued based on the same conditions."

## Solution — launchd LaunchAgent

`bin/cc-install-sweep-agent` writes `~/Library/LaunchAgents/
com.farishijazi.cc-notify-sweep.plist` (`StartInterval=30`, `RunAtLoad`,
`ProcessType=Background`) whose `ProgramArguments` is the `cc-sweep` sitting next to
the installer. launchd runs it every 30s independent of any Claude session.

Crucially, a tick is **just `cc-sweep`** — so it inherits every guard for free:
throttled ≤1/10s, skipped while typing (live OS HID read), and **queued** (single
mkdir-locked drainer) if blocked. That's exactly "fire every 30s, queued on the same
conditions". Overlapping ticks coalesce (second tick can't grab the lock → exits).

The plist points at `$here/cc-sweep` (installer's own dir), so:
- On the dev machine the user runs it from `~/Projects/cc-notify/bin` → points at the
  always-latest dev copy (stable path, no redeploy needed for the agent itself).
- A marketplace user runs it from the installed version dir; **re-run after a plugin
  update** to re-point (same model as `cc-install-editor-extension`).

`--uninstall` boots it out + deletes the plist. Errors → `/tmp/cc-notify/sweep-agent.log`.
Interval override: `CC_SWEEP_AGENT_INTERVAL`.

## Why not a hook / cron / shell loop

- A plain `while sleep 30` loop dies with its terminal and isn't supervised.
- `cron` has 1-min granularity min and a sparse env.
- launchd is the native, supervised, survives-logout/reboot answer with sub-minute
  `StartInterval`.

## Test

Installed, confirmed `launchctl list` shows the label (pid, exit 0). Watched
`/tmp/cc-notify/.sweep` mtime across intervals with no typing → it advanced ~every 30s
on its own (agent-driven), e.g. `…925 → …955` (exactly +30s). The keyboard guard means
a tick while typing queues instead of firing — invisible during active work.

## Deploy

`.claude-plugin/plugin.json` 1.7.6 → **1.7.7**. The agent is already loaded on this
machine (installed during this session). Marketplace deploy still needed for the hook
triggers + so a marketplace user can find `cc-install-sweep-agent`: commit → push
`main` → `/plugin marketplace update farishijazi-plugins` → `/reload-plugins`.
